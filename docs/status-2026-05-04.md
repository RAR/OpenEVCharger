## C. TLV protocol surface

Spec called for 12 commands + 6 events. Shipped: **20 commands + 19 events** (RFID +
lifetime/build/device-id/auth-config extensions). STATE_REPORT is at 56 B =
TLV_PAYLOAD_MAX cap; future fields need a payload-cap bump or a side-channel event.

## D. Open carve-outs

- `FAULT_DIODE_CHECK` — enum-only, blocked on the bench 5-point negative-half cal.
- `FAULT_CC_OUT_OF_RANGE` — decoder live; detector raise gated on
  `OPENBHZD_CC_DETECTOR=1` until bench cal of the OEM CC divider
  (bench unit reads PA7 raw≈12 with no plug, M2 sanity recorded raw=4095,
  topology inconsistent until characterised).
- `FAULT_HARD/SOFT_OC`, `FAULT_RELAY_WELD/STUCK_OPEN` — cal-gated on BL0939 IA scale.
- `FAULT_GFCI_SELF_TEST` — wired (`gfci_self_test()` polarity-agnostic CAL pulse, `self_test_gfci_cal()` boot wrapper); raise gated behind `OPENBHZD_GFCI_CAL_SELF_TEST=1` until bench validates the PE3-CAL→PE2-sense path on this PCB. Live GFCI path now bench-validated (2026-05-05, current injection on PE → external module trip → PE2 LOW → FAULT_GFCI in 3 ticks → EVSE→FAULT → contactor force-open), so only the self-test pulse needs scope validation.
- Relay actuate-readback gated by `OPENBHZD_RELAY_ACTUATE_SELF_TEST = 0`.
- ~~cp_low ETSIC trigger swap doesn't fire on this GD32F205 — try ADC1-dedicated next.~~ **Fixed:** ADC1-dedicated path with TIMER0 CH3 trigger and CCxE=ENABLE captures cp_low cleanly (12000 mV in idle_high as expected).
